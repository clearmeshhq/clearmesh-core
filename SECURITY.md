# Security Policy

Report security issues privately to security@clearmesh.net.

Do not open public issues for vulnerabilities, credential exposure, authorization bypasses, encryption failures, signed URL leaks, or plaintext data exposure. Include affected versions, reproduction steps, and impact.

ClearMesh encrypted repositories use client-side encryption. Passphrases and derived keys should not be sent to ClearMesh services. Public unencrypted repositories intentionally store readable file chunks and do not require a key.
